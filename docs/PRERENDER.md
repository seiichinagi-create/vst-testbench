# PRE-RENDER エンジン — リアルタイム処理を捨てるという設計

> **思想: 未来が既知なら、スループットをレイテンシーに変換できる。**
> 動画プレイヤーの先行デコードや Reaper の Anticipative FX processing と同族。ここではその「RAM全量版」をやる。

## なぜリアルタイムを捨てるのか

従来のVSTホストは、オーディオコールバックの32〜512サンプルという小さなブロック単位に
全ての処理を押し込む。ブロックが小さいほどレイテンシーは減るが、1ブロックあたりの
締切も短くなり、重い処理(高精度EQ、畳み込み、STFT系、ニューラル系)は破綻する。
**リアルタイム制約こそが処理品質の天井**になっている。

ソースが「ファイル + FXチェーン」なら未来は全部わかっている。ならば:

1. FXチェーンの**オフライン複製**(getState/setState同期)を背景スレッドで回し、
   ファイル全体を**大きなメモリブロック単位**で先行レンダリングしてRAMキャッシュに書く
2. オーディオスレッドは**キャッシュを流すだけ** — ASIO 32サンプルでも原理的に破綻しない
3. 再生がレンダーカーソルに追いつけば、音飛びでなく**動画バッファリング式に停止して待つ**

VST処理はオーディオクロックの締切から解放され、大ブロックのオフライン処理として
実時間より速く走ればよい。制約は各段のレイテンシーではなく**合算スループット > 1x** だけ。

## 進化の記録(2026-07-08、1日で v0.3.0 → v0.6.0)

| 版 | 内容 |
|---|---|
| v0.3.0 | PRE-RENDERモード初版。全量先行レンダリング+バッファリング式ストール。ノブ検知→600msデバウンス→自動再レンダリング。プラグインレイテンシー補償(先頭discard)。ファイル→FX経路限定(MIDIは対象外と決定) |
| v0.4.0 | カーソル優先再レンダリング。「カーソル−2秒ウォームアップ→カーソルから書込→末尾→先頭に回り込み」の2パス化で、ノブ変更の反映を数秒→サブ秒に短縮 |
| v0.5.0 | クイック更新+ジャニター。ノブ変更時はカーソル先の短い窓だけ焼いて即終了(全曲焼きの1/10以下の仕事量)。ノブが3秒静まったらジャニターが全量2パス掃除 |
| v0.6.0 | **未来切替点方式**。書込開始点を再生ヘッドの未来(実測レンダー速度から適応リード)に置き、到着時には新音が敷設済み。HLSの次セグメント境界切替と同型で、**切替はサンプル精度で確定的**(追い抜きレース・裂け目リスクを解消)。ループ時は末尾越え切替点を先頭にラップ |

実装: `Source/RenderAheadEngine.h`(RenderCache / CacheAudioSource / Engine)、
キャッシュ上限150Mサンプル(≈52分/1.2GB)。ソース段は `Source/FilePlayerProcessor.h`。

## この仕組みが本当に目指すもの — GPU駆動プラグイン環境

キャッシュが全段をオーディオクロックから切り離すことで、次が成立する:

- **CUDA適応させた自作エンジンを乗せる**: 市販VST3はバイナリ固定でGPU実行不可だが、
  自作FX群はソースがあるのでCUDA/torch移植できる。「リアルタイム放棄」を前提に設計した
  STFT系・ニューラル系の重いエンジンが、この仕組みの上で**実質リアルタイム試聴**できる
- **GPU→CPUの異種混成パイプライン**: GPU(自作CUDAエンジン)→CPU(市販VST3マスタリング段)
  →キャッシュ→極小バッファ再生。市販VSTはGPUに載る必要がなく
  「実時間より速く処理できる」だけでよい(大ブロックオフラインなら容易)
- **ネットワーク越えすら可能**: LAN上のGPUマシンをレンダーサーバーにして
  ファイル→リモートGPU→ローカルCPU→キャッシュ、という構成も同じ原理で成立する

**レイテンシーは死んだ。スループットだけが真実。**

次のステップは、この試作環境の上に乗せる最初のCUDA自作エンジンの開発。

## 既知の注意点

- ASIOドライバが稀に不規則挙動(デバイスリフレッシュで回復)。対策としてレンダースレッドの
  低優先度化と「Reset ASIO」ボタン(close+restart・再生状態維持)を実装済み。再発観察中
- label系の進捗表示: 「Quick update...」→「Window fresh - full sweep when knobs settle...」
  →「Updating from cursor...」→「Cache ready (N.Nx realtime)」
