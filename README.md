# VST TestBench

VSTプラグインの動作確認・ミックス確認を高速化するための軽量VST3ホストである。JUCE 8(8.0.13)スタンダードアロンアプリケーションとして実装する。DAWの重い機能を捨て、「MIDIスイッチング + VST再生」に特化する。

## 概要

本アプリケーションは当初、単一VSTの迅速な音出し確認を目的として開発されたが、以後の拡張により「レイテンシーをスループットへ変換する」実験基盤へ発展した。到達点として、GPU(Python/CUDA)→ VST3(CPU)→ RAMキャッシュという異種混成パイプラインが成立している。

## 主要機能

| 機能 | 内容 |
|---|---|
| ソース段3系統 | ①Live入力(ASIO実機経路) ②VSTiプレイバック(MIDI in→VSTi→FX) ③音楽ファイルプレイヤー(WAV/FLAC/MP3/OGGネイティブ・m4a/AAC/OPUS等はffmpegフォールバック) |
| PRE-RENDER | FXのオフライン複製によりファイル全体をRAMへ先行レンダリングし、再生はキャッシュを流すのみとする。ノブ変更はカーソル先窓の即時焼き直し+静穏後の全量掃除(ジャニター)で追従する |
| バックエンド自動切替 | ファイル再生=WASAPI / Live・VSTi=ASIO をソースモードに連動して自動選択する |
| GPU FX | gpufx常駐ワーカー(TCP)によるファイルのGPU処理。describeスキーマからパラメータUIを動的生成する |
| MIDIバウンス | ドラレコ式MIDI自動保存(無入力10秒で自動保存)→ VSTiオフラインバウンス → GPU/VST/キャッシュ経路へ合流 |

思想と進化の記録は [docs/PRERENDER.md](docs/PRERENDER.md) を、ビルドおよび使用方法は [BUILD.md](BUILD.md) を参照のこと。

## 設計上の要点

- **キャッシュが全段をオーディオクロックから切り離す**ため、各段の制約はレイテンシーではなく合算スループット(実時間比1超)のみとなる。CUDA非対応の市販VST3も最終段に載せられる。
- 自作VST3のオフライン複製に際しては、JUCEラッパーが自動付加する実バイパスパラメータがstateコピーで複製へ運ばれる既知の問題に対処済である(state同期直後に明示的に0化)。

## ビルド方法

Visual Studio 2022 および CMake 3.22 以上を要する。

```
cmake -S . -B build -G "Visual Studio 17 2022" -A x64
cmake --build build --config Release
```
